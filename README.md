# 世界游戏

一个类似 Minecraft 的极简沙盒模拟游戏，用于测试 AI 智能体。

## 运行方法

人类控制客户端（Raylib 可视化）：

```bash
python -m interactive.human.rayplayer
```

AI 可视化演示（回放 example.py 序列）：

```bash
python visual_example.py
```

冒烟测试（开发用）：

```bash
.venv\Scripts\python.exe example.py
```

## 游戏规则

### 世界

离散世界，离散移动，离散时间。每 tick 执行世界运算。

坐标格式为 `(x, y, z)`，其中 **x 轴正方向为东**，**y 轴正方向为上（高度）**，**z 轴正方向为南**。世界原点固定在角落，为 `(0, 0, 0)`，坐标范围 `0 ≤ x, y, z ≤ 4`，世界大小 5×5×5。

玩家初始位置 `(0, 0, 0)`，朝向东，物品栏只有一个树苗。

### 树苗生长

每 tick，所有满足条件的树苗有 **10% 概率**长成一棵树：

- 树苗必须位于 `y = 0`（地面层）
- 树苗所在竖列（相同 xz）及四个水平邻列中不能有任何其他方块
- 树干高度从 2、3、4 格中等概率随机选取（即各占总生长概率的 1/3）
- 生长后：树苗消失，从 `y=0` 向上放置指定数量的木板，顶部放 1 格树叶
- 若最终执行时条件不满足（如另一棵树同时长大堵住了空间），生长取消
- 各树苗独立计算，互不影响

### 物品

| 物品 | 最大堆叠 | 说明 |
|------|---------|------|
| 树苗 | 8 | 只能在 `y=0` 放置 |
| 木板 | 8 | 可放置 |
| 木斧 | 1 | 不可放置，挖木板速度翻倍 |

### 方块

| 方块 | 破坏时间/t | 掉落物 | 掉落数量 | 有实体碰撞 |
|------|----------|------|--------|---------|
| 树苗 | 1 | 树苗 | 1 | 无 |
| 木板 | 4（手持木斧时 2）| 木板 | 1 | 是 |
| 树叶 | 1 | 树苗 | 1–3（等概率） | 是 |

"有实体碰撞"的方块会阻挡玩家移动。方块不受重力影响。

### 合成

| 配方 | 材料 | 耗时 |
|------|------|------|
| 木斧 | 3 块木板（物品栏任意位置合计） | 1 t |

合成失败条件：木板不足 3 块，或物品栏无空间容纳木斧。

### 玩家

- **高度**：2 格（脚格 + 头格）
- **移动速度**：1 格/t，只能向当前朝向方向前进
- **台阶跨越**：前方同高被实体方块阻挡、但高一格（step up）无障碍时，自动上台阶，耗时与平地移动相同（1 t）；前方两格高均有实体方块则无法移动
- **物品栏**：9 格（槽 0–8），同一格只能放同类物品的一个堆叠
- **活跃槽**：物品栏中当前选中的槽位（默认 0），主手物品即该槽物品

### 玩家行动（Intent）

**0 tick 行动**（免费，可与 1 tick 行动合并在同一刻提交）：

| 行动 | 说明 | 是否中断挖掘 |
|------|------|------------|
| 转向（TurnIntent） | 切换朝向，东/南/西/北任意方向 | 是 |
| 等待（NoIntent） | 什么都不做 | 是 |
| 切换活跃槽（ChangeActiveSlotIntent） | 切换主手槽位 0–8 | **否** |

**1 tick 行动**：

| 行动 | 说明 | 是否中断挖掘 |
|------|------|------------|
| 前进（WalkIntent） | 向朝向方向走一格（含台阶跨越） | 是 |
| 挖掘（DigIntent） | 挖指定（或当前）方块 | — |
| 放置（PlaceBlockIntent） | 在指定位置放置主手物品对应方块 | 是 |
| 合成木斧（CraftAxeIntent） | 消耗 3 木板合成木斧 | 是 |
| 丢弃一个（DropOneIntent） | 丢弃活跃槽（或指定槽）1 个物品 | 是 |
| 丢弃整堆（DropStackIntent） | 丢弃活跃槽（或指定槽）全部物品 | 是 |
| 交换物品槽（SwapInventoryIntent） | 交换任意两个槽的内容 | 是 |

丢弃的物品直接消失，不产生世界实体。

### 挖掘范围

玩家同时只能挖一个方块；切换目标会重置进度。可挖的 5 个位置（相对于玩家脚部坐标 `(x, y, z)`，朝向偏移为 `(dx, 0, dz)`）：

| 位置 | 坐标 | 说明 |
|------|------|------|
| 正前方（脚高） | `(x+dx, y, z+dz)` | |
| 正前方（头高） | `(x+dx, y+1, z+dz)` | |
| 脚下 | `(x, y−1, z)` | 挖后玩家会因重力下落 |
| 头顶 | `(x, y+2, z)` | |
| 前下方（搭桥前） | `(x+dx, y−1, z+dz)` | 前方向下一格 |

挖掘需要消耗若干 tick（见方块表）。最后一刻挖完时，按概率将掉落物加入物品栏；物品栏满时超出部分静默丢弃，但挖掘动作仍算完成。

### 放置范围

可放置的 5 个位置（同样相对于玩家脚部）：

| 位置 | 坐标 | 特殊条件 |
|------|------|--------|
| 正前方（脚高） | `(x+dx, y, z+dz)` | |
| 正前方（头高） | `(x+dx, y+1, z+dz)` | |
| 自身位置（垫高） | `(x, y, z)` | 非树苗需头顶 `(x, y+2, z)` 无障碍；树苗仅允许 y=0 |
| 前下方（搭桥） | `(x+dx, y−1, z+dz)` | |
| 头顶 | `(x, y+2, z)` | |

放置要求：目标格为空、目标格在世界范围内、且与已有方块六面相邻（y=0 层无需相邻）。树苗只能放在 `y=0`。放置消耗主手槽 1 个物品。

## 游戏玩法详解

本章从代码层面系统整理游戏中所有的设计思考，便于理解整体机制。

### 时间模型

游戏时间以"刻（tick）"为最小单位推进。每刻流程：

1. 收集玩家提交的意图队列，按顺序消耗直到总 tick_cost 累计达到 1.0 为止
2. 收集世界自发变化（树苗生长等）
3. 从所有独立随机事件的笛卡尔积中按概率采样，得到本刻唯一确定的结果
4. 执行所有 mutation，每个 mutation 执行前校验条件是否仍成立
5. 运行物理模拟（重力、碰撞、边界）
6. tick 计数 +1

玩家每刻最多贡献合计 tick_cost = 1.0 的意图。0 tick 意图（转向、等待、切换槽）可免费附加在 1 tick 意图之前的同一刻提交。

### 物理规则

物理在每刻末尾迭代执行（最多 32 次迭代）直到世界稳定：

**重力**：玩家若脚下 `y-1` 无有实体方块（且 `y > 0`），则持续向下掉落，直到落到某有实体方块上方或 `y = 0`（地面层）。

**碰撞解决**：若玩家与有实体方块发生重叠（例如树苗在玩家脚下长成、或玩家走入了刚生成的树中），系统用 BFS 搜索最近的可容纳玩家的空位并将玩家传送过去。

**边界夹紧**：玩家坐标被强制限制在 `x, z ∈ [0, 4]`，`y ∈ [0, 5 − height]` 范围内。

**关键推论**：
- 玩家在自身脚部坐标放置有实体方块 → 方块放好 → 物理执行 → 玩家与方块重叠 → 碰撞解决 → 玩家被顶到方块上方（即被垫高 1 格）
- 树苗在玩家站立格旁边长成 → 若树干占据玩家所在格 → 碰撞解决 → 玩家被弹到最近空位
- 挖掉脚下方块 → 重力 → 玩家下落一格

### 挖掘计时

开始挖掘的那一刻本身计为 1 个单位的进度：
- 树苗（1.0 tick）和树叶（1.0 tick）：第 1 刻即挖完，掉落物当刻入袋
- 木板（4.0 tick）：挖 4 刻完成，每刻提交一次 DigIntent
- 木板 + 木斧（2.0 tick）：挖 2 刻完成
- 切换目标或执行会中断挖掘的意图 → 进度清零，无法继续

### 物品栏机制

- 9 格物品栏（槽 0–8），活跃槽默认 0
- 物品入栏优先堆叠到同类型已有槽，满后才放入空槽
- 超出上限的物品静默丢弃（不影响动作本身的成功）
- 切换活跃槽（ChangeActiveSlotIntent）不消耗 tick，不中断挖掘
- 交换两槽（SwapInventoryIntent）消耗 1 tick，中断挖掘
- 丢弃物品直接从物品栏移除，不产生世界中的掉落物实体

### 树木生长细节

- 每棵树苗独立参与概率判断（各 10% 生长，90% 不动）
- 条件检查分两阶段：判断是否尝试生长（tick 开始时）+ 执行时再次确认（防止同 tick 内其他事件改变了条件）
- 相邻两棵树苗可能同时抽中生长，若它们的生长范围互相冲突，执行时 check_conditions 失败的那棵直接跳过
- 树叶只有 1 格，位于树干正上方

### 放置与垫高

在自身坐标（脚部）放置有实体方块是合法的"垫高"操作，但有前提：
- 玩家头顶 `(x, y+2, z)` 必须无方块遮挡（否则垫高后玩家无处容身）
- 垫高本身是放置动作（1 tick）+ 物理模拟的联动结果，并非特殊的移动意图

树苗是无实体方块，放在自身脚部不触发垫高，且只允许在 `y=0` 放置。

### 台阶跨越

玩家走向前方：
- 前方同高（`y`）无障碍 → 直接移动
- 前方同高有实体方块，但前方高一格（`y+1`）两格高空间（`y+1`, `y+2`）均无障碍，且 `y+1` 处有实体方块作为台阶 → 自动上台阶至 `y+1`，耗时同样 1 t
- 否则（两格高均堵） → 移动失败，原地不动，仍消耗 1 t

### Mutation 概率模型

每 tick 的世界变化被建模为"Mutation 组序列"（MutationGroupSequence）：

- 每个 **MutationGroup** 代表一个独立随机事件（如"树 A 是否生长"），内含若干互斥的 mutation 候选及其权重
- MutationGroupSequence 是多个独立 MutationGroup 的笛卡尔积
- 采样时对每个 group 独立按权重抽取一个 mutation，组合成本刻唯一的 MutationSequence
- 每个 mutation 在执行前必须通过 `check_conditions` 验证，条件不满足则跳过（不报错）

这一设计使 AI Agent 可以精确计算每种世界状态的概率，而无需真正执行游戏逻辑。

## 游戏机制（开发笔记）

### Mutation 机制

所有世界变化均通过 Mutation 提交，禁止直接修改 world 状态。每个 Mutation 体现一种可能的变化及其概率。每 tick 产生的所有 Mutation 组合成 MutationGroupSequence，例如玩家挖掉树叶的那一刻，地面上还有两个树苗等待生长：

```
[
    [(挖掘完成, 1.0)],
    [(物品栏+1 树苗, 0.33), (物品栏+2 树苗, 0.33), (物品栏+3 树苗, 0.33)],
    [(树A不动, 0.9), (树A长成2格, 0.033), (树A长成3格, 0.033), (树A长成4格, 0.033)],
    [(树B不动, 0.9), (树B长成2格, 0.033), (树B长成3格, 0.033), (树B长成4格, 0.033)]
]
```

各组互相独立，最终采样得到一组确定的 MutationSequence 并执行。

### 物理

玩家受重力影响，方块不受重力影响。每 tick 末尾物理迭代直至稳定：

1. 对所有玩家施加重力（循环下落直到有支撑或到达 y=0）
2. 对所有玩家解决碰撞（BFS 找最近可容纳位置）
3. 对所有玩家夹紧到世界边界内

若 32 次迭代后世界仍不合法，抛出异常。
