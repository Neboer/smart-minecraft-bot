# 世界游戏

一个类似minecraft的极简沙盒模拟游戏，用于测试。

## 世界规则

离散世界，离散移动，离散时间。
每tick执行世界运算，规则书里所有a-b都代表a<=x<=b的均匀整数分布。

## 树苗生长

每t，近邻四格和垂直方向上均无任何方块、且不与玩家坐标相同的树苗方块有10%概率变成树，就是垂直2-4格木板+1格树叶的结构。

## 物品

树苗：只能作为高度0的方块放置，物品堆叠8
木板：可放，物品堆叠8
木斧：不可放，物品堆叠1。

## 方块

| 物品名字 | 破坏时间/t | 掉落物 | 掉落数量 | 有无实体 |
|--------|----------|------|--------|--------|
| 树苗 | 1 | 树苗 | 1 | 无 |
| 木板 | 4 | 木板 | 1 | 是 |
| 树叶 | 1 | 树苗 | 1-3 | 是 |

## 合成

木斧：需3木板 消耗2t

## 玩家
0. 玩家高度为2，高度为2的空间内有障碍时玩家无法跨越。
1. 玩家移动速度 1格/t
2. 玩家物品栏有10格（0-9），其中第0格为主手，每格最多可以放一堆叠同类的物品，不同的物品必须放在不同格中。
3. 玩家可以任意交换物品栏物品的位置，不消耗t。
4. 玩家主手是木斧时，挖掘木板速度翻倍；玩家主手是物品时，才可能可以放置物品。

玩家只能破坏自己面前与自己同高或高度+1、或自己脚下的方块（玩家破坏自己脚下的方块时，玩家高度-1）。
玩家只能同时破坏一个方块，否则会重置破坏并重新计算破坏时间。
玩家只能放置自己面前与自己同高或高度+1、或自己脚下的方块（在自己脚下放置有实体方块时，玩家高度+1，相当于垫起，前提是玩家头顶必须没有方块遮挡才能垫起；放无实体方块时，玩家高度不变），并且必须与已经存在的方块相邻，或高度=0的第一层上放置，不能悬空放置。
玩家放置任何方块都需要消耗1t。

1. 玩家具有朝向，朝向分为东南西北四个方向，转到任何方向都要1t，玩家只能向朝向的方向移动.

## 初始

坐标格式为(x,y,z)，其中x轴正方向为东，y轴正方向为南，z轴正方向为上。
世界原点固定在一个角落，为(0,0,0)，坐标范围为0<=x,y,z<=4。
世界大小5\*5\*5，中心坐标为(2,2,2)。
玩家初始位置为(0,0,0)，朝向东，物品栏内只有一个树苗。

## mutation机制

上述代码的问题是，random把所有的随机都直接计算出来了，最后的world变化成的目标结果是唯一的。我希望程序的结构可以遍历所有可能性，而又不失上述编写方法的便利性。
请给World的每个独立变化增加Mutation功能，代替random直接决定随机状态（无random也要mutation，只是概率为1），每个mutation体现了world的一种可能的变化及这种变化的可能性，每个tick执行之后，会产生一个完整的mutation组组成的列表来表示world的所有可能状态及其概率的组合，比如物品栏增减、树木长成与否等。注意mutation中每个变化是互相独立的，比如玩家挖掉树叶的那一刻，地面上也有两个树苗等待长大，那么这一刻里四个mutation组互相独立：
[
	[(玩家挖掘位置树叶方块消失, 1)],
    [(物品栏增加1树苗, 0.33), (物品栏增加2树苗, 0.33), (物品栏增加3树苗, 0.33)],
    [(树A没有长大, 0.9), (树A长大成2格树, 0.1), (树A长大成3格树, 0.1), (树A长大成4格树, 0.1)],
    [(树B没有长大, 0.9), (树B长大成2格树, 0.1), (树B长大成3格树, 0.1), (树B长大成4格树, 0.1)]
]
注意玩家行为也是一个mutation组（现在玩家行为是直接影响世界的，请你修改为提交Mutation），只是确定的——所有世界变化必须通过mutation组提交，不允许擅自修改world，最后Game（当前代码不存在，你需要再抽象一层Game）中每个tick会收集这一tick中产生的所有mutation组，与玩家这一tick输入的Mutation结合（玩家不是每一tick都必须行动的，不行动也是一种行动，比如玩家挖掘木头可能挖掘多个tick），形成一个WorldMutations，每个tick执行之后会返回完整的WorldMutations，然后Game对象会“SelectMutation”，按概率生成出一组MutationSequence表示所有可能的变化的集合，提交给“ExecuteMutation”，执行MutationSequence，将世界变换到下一个确定的状态。
请你保持程序文件和类文件结构的清晰干净，拆分合理。
