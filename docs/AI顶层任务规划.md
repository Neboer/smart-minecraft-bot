# AI顶层逻辑推理算法

AI需要一个“大脑”，决定它应该去哪里做什么。

同一时刻，AI要么正在种树，要么正在砍树，要么正在等待。其中任务包括赶路的时间。

那么AI应该什么时候去哪里种树，什么时候去哪里砍树，什么时候等待呢？

这个AI顶层逻辑推理算法就是为了解决这个问题而设计的。以下是算法具体流程：

在顶层搜索时，AI进行如下考虑：

## 顶层搜索

AI总任务分成三种：
1. 种树
2. 砍树
3. 等待

AI的宏观状态：
1. 当前游戏tick
2. 当前物品栏木头总数，树苗总数
3. 场上所有树的：
   1. 位置坐标
   2. 大小
4. 场上所有有效树苗的：
   1. 位置坐标
   2. 种下时的tick数
5. 距离玩家最近的可以放树苗的位置坐标

### AI种树：
AI种树，每次都在距离玩家尽可能近的地方种一棵树。
需求：
1. 树苗总数>0
2. 距离玩家最近的可以放树苗的位置坐标存在
估算时间：
1. 寻路到距离玩家最近的可以放树苗的位置坐标的时间
2. 切换主手时间，放置树苗时间
获得木头量：
0
变化：
1. tick+估算时间
2. 树苗总数-1
3. 场上树苗增加一个到距离玩家最近的可以放树苗的位置坐标，种下时的tick数为当前tick+估算时间。
4. 进行树的成长计算，到期望个tick树就会长成。
5. 玩家位置变化，距离玩家最近的可以放树苗的位置坐标重新计算。
输出：
“向距离玩家最近的可以放树苗的位置坐标种树。”

### AI砍树：
AI砍树，每次都砍掉距离玩家最近的树。
需求：
1. 场上有树。
估算时间：
1. 寻路到树的位置坐标的时间
2. 砍掉这棵树的总时间。估算方法：树的大小\*挖掘时间+(树高度-1)\*垫脚\*挖掘时间
获得木头量：
树的大小
变化：
1. tick+估算时间
2. 场上树减少一棵
3. 物品栏木头总数增加树的大小
4. 进行树的成长计算，到期望个tick树就会长成。
5. 玩家位置变化，距离玩家最近的可以放树苗的位置坐标重新计算。
输出：
“向距离玩家最近的树的位置坐标砍树。”

### 等待：
AI等待，当没事可做的时候，AI选择等待。
需求：
随便。
估算时间：
1tick
获得木头量：
0
变化：
1. tick+1
2. 进行树的成长计算，到期望个tick树就会长成。
输出：
“等待。”

AI展开状态图，对上述动作和状态进行广泛搜索，找到一条使 木头增量/时间增量 最大的路径。

## 具体实现

方法有了，比如去砍树，那怎么具体的实现呢？AI需要知道怎么去砍树，怎么去种树。

### 寻路算法：二维粗略寻路

将世界地图中所有超过2的高度边界表示为二维迷宫中的墙，它们将地图划分成不同的可及域。找出地图中当前的所有可及域，计算凝聚图。

在大场景搜索时，将MC地图根据高度墙拆成不同的可行域，然后将这个可行域组成的着色地图变成有向图，每个可行域都是其中一个节点。 节点之间的边的方向就是高度梯度的方向，节点相邻则有边，边的权重就是逆高度方向跨过两个可行域的难度，也就是边界处最低的高度差，此时边界处高度差最低的点，是两个节点之间最容易跨越的点，叫作两个节点之间的门户点（portal）。

将玩家和目标点所在的可行域的节点找出。在图中搜索出一条边：从玩家节点指向目标节点，需要反向的边权重之和最小，或同时考虑路径本身长度评分，综合得出一条经过最少的可行域，花最小力气垫高的路线。

机器人希望抵达目标，则找到自己所在与目标所在的可及域。在相同可及域里通过A\*算法抵达目标位置。

在可及位置中走到目标位置附近，然后搜索如何改变周围方块能更多的让可及位置里出现距离目标位置更近的位置（最近可及点距目标的距离/代价），

### 挖树算法：三维精细抵达

AI启发式搜索一种方法垫高自己+砍掉整树，然后再把垫脚砖全挖掉。由于在mc里的树往往会非常复杂，所以这个算法不应该追求固定的解，应该使用启发式搜索。**每次找到一个距离自己最近的树的方块作为目标，然后逐渐挖掉所有**。唯一需要搜索的就是到底如何挖掉“距离自己最近的树的方块”。

展开状态图，允许动作序列：挖掉方块、垫高、移动、转向、放置方块。在世界状态里，应该有一个“可及位置”列表，表示如果AI不改变地形，AI可以让自己的x,y到哪个位置。

大规模寻路与小规模寻路算法不同。大规模寻路算法采用计算可及域+最近点的方法，利用A\*接近目标；小规模寻路直接展开状态图，抵达目标附近就算成功。一般来说为了挖掉树，AI需要建一座垫脚塔，垫脚塔其实是新的树，等AI确认将一株树挖完之后，AI就把垫脚塔挖掉.

搜索永远开始于距离目标位置最近的可及位置，然后开始到下一批距离目标不那么近的位置，然后以此类推……
玩家到可及域上这样一点，然后挖掉或放置周围1格包围内的方块，看看是否可以获得更近的可及位置。如果有，则走到那个位置，继续这个过程。
如果没有，则搜索挖掉或放置周围1格包围内的2个连续方块是否可以获得更近的可及位置，如果有，则走到那个位置，继续这个过程。
在一般的MC机器人里，这样的挖掘或放置一般可以让AI获得更近的可及位置。
但我们很快可以想到一个反例：（B是实心树方块，G是目标位置，P是玩家位置，C是另外一点）
```
  B B G
  P B B
  P B B
  B   B C
  - - -
```
在这种情况下，玩家挖去面前的B B，也不能获得更近的可及位置。怎么办？没什么好办法，我们宣布此状态无解，AI放弃可行域里最近的位置“当前位置”，去寻找次近的位置，继续这个过程。

AI来到地面上，绕一圈来到G所在柱子的右边C点，这里是次近位置（虽然略有点远），在这里展开搜索。AI很快发现如果垫高可以获得更近的可及位置，于是AI垫高，继续这个过程，最终让目标位置可及，GOAL！

### 什么是树

连续的竖直几个方块是树，哪怕它是悬空的。树的大小是竖直连续几个方块的数量。
