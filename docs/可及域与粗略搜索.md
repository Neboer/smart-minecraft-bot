# 可及域粗略搜索跨越算法

AI想要从一点抵达另外一点，这两点在不同的封闭可及域里，距离比较远。

本算法提出一种简单思路，将世界变为封闭可及域组成的有向带权连通图，并在里面找到一个从当前封闭可及域到目标封闭可及域的、需要最少地形修改+走路时间的路径。

找到路径后，AI利用A\*在当前封闭可及域里做迷宫搜索，走到一可及域的门户，修改地形跨越到下一个可及域，重复执行直到抵达目标。

一个封闭可及域中的寻路与A\*算法完全一样。

## 封闭可及域

在《三维精确抵达算法》中定义了什么是可及域（Region），它是一个玩家从当前位置，在完全不改变世界的前提下，能够通过纯移动到达的所有位置的集合。

可及域与玩家位置相关，地图相同，但玩家位置不同，可及域也不同。玩家在高处可及域就大一些，玩家在低处可及域就小一些。

封闭可及域的定义是：如果两个站立位置能够通过纯移动互相到达，则属于同一个封闭可及域（Closed Region）。封闭可及域实际上等价于站立图上的一个强连通分量。

封闭可及域与玩家位置完全无关，地图相同，封闭可及域也完全相同。封闭可及域是地图的一个固有属性。因此，通过查找固定地图的所有封闭可及域及其之间的连接关系，我们就可以构建一个封闭可及域图（Closed Region Graph），在这个图上进行搜索，找到从玩家所在封闭可及域到目标所在封闭可及域的路径，这就是“可及域粗略搜索跨越算法”的核心思想。

## 封闭可及域图

封闭可及域图（Closed Region Graph）是一个有向带权连通图，其中：
- 每个节点表示一个封闭可及域
- 每条边表示两个封闭可及域之间的连接关系，边的方向表示容易从一端封闭可及域到达另一端的，并且不需要跨过其他的封闭可及域。
- 每条边的权重表示从一个封闭可及域跨越到另一个封闭可及域所需的 最小行走成本 + 最小地形修改成本

## 利用封闭可及域图寻路

在封闭可及域图中搜索：

- 起点封闭可及域是玩家当前所在的封闭可及域，终点封闭可及域是目标所在的封闭可及域。
- 从起点域到终点域的每条路径，都表示为了抵达目标可行的封闭可及域级路线。
- 其中最短的路径（权重最小的路径）表示抵达目标的最优封闭可及域级路线。

找到最优封闭可及域级路线后，AI在两个可及域间使用可及域间跨越算法进行跨越。

## 与跨越算法协同工作

在进行可及域跨越的时候，可能会出现可及域的变化，导致规划好的路线出现变动。这个问题难以解决，直接限制了本方法 大搜索 + 小搜索 的适用范围不适用小范围的跨越问题。

在小范围跨越问题中建议直接用小搜索。本粗略搜索方法本质上是提供距离较远的问题，较远的评价指标是：除了正在被抵达和玩家当前所在的可及域这两个封闭可及域之外，封闭可及域图中剩下的所有其他封闭可及域都不会随玩家穿越的行为而变化。、

但实际上在大层次穿越的时候，可能会有许多零碎的小可及域。因此每次玩家成功穿越之后就再搜索更新一下吧！
