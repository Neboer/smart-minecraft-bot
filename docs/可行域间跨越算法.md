# 可行域间跨越算法

当MC机器人已经离一点比较近的时候，如何在复杂的地形中，合理的考虑地形，挖掘、放置方块，不断的逼近目标，直到精确抵达目标点呢？本算法就是为了解决这一具体问题。

## 问题

机器人在一点P，目标在一点G，P与G之间的距离不远，但之间可能有任意障碍。

机器人可以挖掘放置方块，可以四向移动，它最终需要让自己的坐标等于G。

## 可及域

可及域（Region）定义

Region 是满足以下条件的最大可能离散的三维区域：

- 玩家身高固定（2格）
- 考虑重力，玩家会从任意高度瞬间掉落到地面上
- 考虑玩家可以走上身前较低的1格方块上
- 玩家会被其他实体方块阻挡
- 不放置和挖掘方块，世界的方块不会改变

在此条件下：

玩家从点P出发，可以在空间中直接通过一条路线抵达的任何区域则此两点属于同一个可及域Region。

比如玩家在高点，那么他所在的高点1格和下方所有的低点都属于同一个可及域。

可及域因玩家所在位置的不同而不同。玩家从高点到低点后，不能再返回高点，因此可及域缩小。

可及域的优势：

玩家在同一可及域之中的移动可以简化为简单的在二维迷宫中的路径规划问题，直接使用A\*算法进行导航。

## 封闭可及域

可及域（Region）是一个玩家从当前位置，在完全不改变世界的前提下，能够通过纯移动到达的所有位置的集合。可及域与玩家位置相关，地图相同，但玩家位置不同，可及域也不同。玩家在高处可及域就大一些，玩家在低处可及域就小一些。

封闭可及域的定义是：如果两个站立位置能够通过纯移动互相到达，则属于同一个封闭可及域（Closed Region）。封闭可及域实际上等价于站立图上的一个强连通分量。

封闭可及域与玩家位置完全无关，地图相同，封闭可及域也完全相同。封闭可及域是地图的一个固有属性。因此，通过查找固定地图的所有封闭可及域及其之间的连接关系，我们就可以构建一个封闭可及域图（Closed Region Graph），在这个图上进行搜索，找到从玩家所在封闭可及域到目标所在封闭可及域的路径，这就是“可及域粗略搜索跨越算法”的核心思想。

举例说明：玩家站在平地唯一的2格高柱子上，此时玩家的可及域是柱子上一格+平地上1格高的所有空间。整个空间包含两个封闭可及域：柱子上1格的封闭可及域和平地上所有1格高的格的封闭可及域。如果玩家到了平地上，那么玩家的可及域就正好等于平地上所有1格高的格的封闭可及域了。

再举一例：玩家在平原上的一个封闭的“火柴盒”木制空间里，玩家需要抵达火柴盒外的一个目标点。玩家的可及域是火柴盒地板上所有1格高的格的集合，世界有三个封闭可及域：火柴盒地板上所有1格高的格的封闭可及域、火柴盒楼顶所有1格高的格的封闭可及域、火柴盒外平原上所有1格高的格的封闭可及域。

## 基本符号

| 符号 | 含义 |
|---|---|
| P | 玩家当前坐标 |
| G | 目标坐标 |
| W | 当前世界状态（所有方块的分布） |
| SP | 玩家当前所在的可及域 |
| SG | 目标 G 所在的封闭可及域 |
| d(SP, SG) | SP 与 SG 之间所有点对中的最小欧氏距离，称为**前沿距离** |
| depth | 当前递归深度 |
| MAX_DEPTH | 最大允许递归深度（建议 6~10，根据世界尺寸和可用材料调整） |

## 方法

### 总体思路

玩家当前所在的位置有一个可及域 SP，目标 G 所在的位置有一个封闭可及域 SG。问题转化为如何从SP抵达SG。

若 G ∈ SP，则玩家与目标已在同一可及域内，直接用 A\* 等路径规划算法在域内导航到 G，完成任务。

若 G ∉ SP，则说明还不能直接抵达目标，需要在当前可及域里走近目标，通过挖掘或放置方块来改变世界，不断改变玩家所处的SP，直到其包含目标G。

算法采用**递归搜索**推进：每一层递归在当前世界状态下，枚举 SP 中的前沿点，对前沿点附近的方块执行动作序列，统一重新计算动作后的新可及域，以新前沿距离 d(SP', SG') 作为评价指标，选择最优分支深入递归，直到成功或达到递归深度上限为止。


### 算法入口

```
Reach(P, G, W, depth):

    计算 SP ← 从 P 出发的可及域（世界状态 W）
    计算 SG ← G 所在的封闭可及域（世界状态 W）

    若 G ∈ SP:
        → 在 SP 内执行 A* 路径规划，导航到 G
        → 返回 SUCCESS

    若 depth ≥ MAX_DEPTH:
        → 返回 FAIL（超过最大递归深度）

    d_cur ← d(SP, SG)    // 当前前沿距离

    → 进入前沿点枚举与动作搜索（见下文）
```


### 前沿点枚举

**前沿点**是 SP 中距离 SG 最近的若干点——它们是当前最有希望通过地形改造接通两个可及域的位置。

前沿点计算的思路是，找出 SP 距离 SG 较近的一系列点集合，将这些点拆成一个个独立的岛。在每个岛里都较为平均的选取距离SG最近的点和一系列较分散的其他点作为前沿点`pi`。具体算法如下：

1. 计算 d_cur = d(SP, SG)
2. 构建 NearBoundary = {p ∈ SP : d(p, SG) ≤ d_cur + 1.5} NearBoundary 是 SP 中距离 SG 最近的一些点的集合。
3. BFS 找 NearBoundary 的连通分量 {R_1, R_2, ..., R_m} 每个分量是一个岛
4. 对每个分量 R_i：
   4a. k_i = max(1, int(K_budget / m))  k_i表示要在该岛中取几个点
   4b. 在 R_i 内，用贪心最大-最小分散选取 k_i 个点
5. 合并所有分量的选取点，返回 F

对每个前沿点 `pi`，将玩家（虚拟地）移动到 `pi`，展开动作搜索.

贪心最大-最小分散算法的实现，

1. 从空间1中找 N = M*K 个距空间2最近的候选点，记为 C = [c1, c2, ..., cN]（按d排序，M一般取3比如说，取3倍的点。）
2. F ← {c1}（先选最近的点）
3. 对 i = 2 to K：
   3a. 对 C 中所有未选点 q，计算 mindist = F中的所有点到 q 的最小距离
   3b. 选取 mindist 最大的 q* 加入 F
4. 返回 F

以上算法就是，找一批从空间1到空间2距离较近的点，将最近的点添加，然后每次选一个与已选点集合距离最大的点加入。

### 动作序列与操纵位置枚举

一次**动作序列**是若干基本动作的有序组合，作为一个整体在副本世界中执行，产生一个新世界状态。基本动作只有两种：

| 动作 | 描述 | 前提 |
|---|---|---|
| `DIG(b)` | 挖掉位置 b 的方块 | b ∈ M，b 有实体方块，b 可被玩家手臂触及 |
| `PLACE(b, item)` | 在位置 b 放置方块 | b ∈ M，b 当前为空，b 相邻有支撑面（或 z=0），库存有 item |

枚举时，将动作序列长度限制为 **1~2 步**，常见有意义的序列模式示例：

1. 在脚下挖一个方块
2. 在身体坐标处填一个方块将自己垫高。
3. 在身边+脚下3\*4=12格（前后左右与 玩家加脚下一格 同三格高）挖一个方块或放一个方块
4. 在头顶上挖一个方块或放一个方块

以上操作对周围的格子完成1次或2次，如果作用2次则不能作用在同一个位置上。

### 统一计算新可及域

对每个（前沿点 `pi`，动作序列 `A`）对，统一执行以下评估流程：

1. **抵达前沿点**：在当前世界 W 中，虚拟的A\*将玩家从 P 导航到 `pi`。
2. **模拟执行**：在世界 W 的副本 W' 上执行动作序列 A，得到新世界状态 W''。
3. **确定玩家新位置**：执行动作后，玩家在 P'。
4. **重新计算两个可及域**：
   - SP' ← 在W''中从 P' 所在的可及域
   - SG' ← 在W''中G所在的封闭可及域
5. **终止判断**：若 G ∈ SP'，则立即返回 SUCCESS，并沿 SP' 内导航到 G。
6. **计算新前沿距离**：d' ← d(SP', SG')。


### 可及域评价与候选分支筛选

以**新前沿距离 d'** 作为当前动作序列的评价指标——d' 越小，说明该动作使两个可及域越接近，越有探索价值。

筛选规则：

- 若 d' < d_cur（当前前沿距离）：**有进展**，将三元组 (P', W'', d') 加入候选列表。
- 若 d' == d_cur：**持平**，作为备选，低优先级加入候选列表。
- 若 d' > d_cur：**退化**，丢弃此分支（若所有分支均退化，则将退化最小的作为兜底保留）。

将候选列表**按 d' 从小到大排序**，优先探索前沿距离下降幅度最大的分支。


### 递归推进

```
candidates ← 按 d' 排序的候选列表（每项为 (P', W'', d', A)）

for each (P', W'', d', A) in candidates:
    result ← Reach(P', G, W'', depth + 1)
    若 result == SUCCESS:
        → 将动作序列 A 记录到总动作规划序列中（前置于后续动作）
        → 返回 SUCCESS

返回 FAIL（所有候选分支均未成功，回溯到上层）
```

若本层所有候选分支均返回 FAIL，本层亦返回 FAIL，触发上层回溯。


### 剪枝与约束

**递归深度限制**：depth ≥ MAX_DEPTH 时直接返回 FAIL，防止无限搜索。MAX_DEPTH 建议设为 6~10，可根据世界尺寸（5×5×5）和库存材料上限动态调整。

**前沿距离单调剪枝**：维护本层已知最优 d_best，若某分支 d' ≥ d_best，则直接跳过，无需递归。

**世界状态去重**：维护已访问的 `(世界方块哈希, 玩家位置)` 集合，对重复状态直接返回 FAIL，避免循环搜索。

**库存约束**：PLACE 动作消耗库存材料，枚举前先检查库存，若材料不足则跳过该动作。


### 算法终止条件汇总

| 情形 | 结果 |
|---|---|
| SP == SG（进入时） | SUCCESS：在域内 A\* 导航到 G |
| 某动作序列执行后 SP' 包含 G' | SUCCESS：在新域内导航到 G |
| depth ≥ MAX_DEPTH | FAIL：当前路径放弃，回溯 |
| 所有候选分支均递归返回 FAIL | FAIL：本层回溯到上层 |
| d(SP, SG) 无穷大（两域完全隔绝） | FAIL：直接放弃，上报无解 |


### 完整流程图

```
Reach(P, G, W, depth)
│
├─ 计算 SP（从P出发）、SG（从G出发）
│
├─ SP == SG?
│   └─ YES → A* 导航到 G → SUCCESS
│
├─ depth >= MAX_DEPTH?
│   └─ YES → FAIL
│
├─ d_cur = d(SP, SG)
│
├─ 枚举前沿点列表 [(p1,d1), ..., (pk,dk)]（SP中距SG较近且分布较分散的一些点）
│
└─ for each 前沿点 pi:
    ├─ 将玩家虚拟移动到 pi
    └─ for each 动作序列 A in 枚举动作(pi, W):
        ├─ W'' = apply(W, A)
        ├─ P' = W''中的玩家新位置
        ├─ 计算 SP'（从P'出发）、SG'（从G出发）
        ├─ G 在 SP' 内部? → 导航到G → SUCCESS
        └─ d' = d(SP', SG')，若 d' < d_cur，加入候选列表（表明此行为使得两个可及域更接近了）

candidates = 按 d' 升序排列的候选列表

for each (P', W'', d', A) in candidates:
    result = Reach(P', G, W'', depth + 1)
    if result == SUCCESS:
        → 记录动作序列 A → 返回 SUCCESS

返回 FAIL
```


### 备注

本算法非常适合用于解决两个距离不远的可及域之间的跨越问题。如果两域相距远，需要跨越许多的其他封闭可及域才能抵达，那么应该先将所有封闭可及域制作成凝聚图，找到抵达目标的最优封闭可及域级路线，再在每两个封闭可及域之间使用本算法进行跨越。但这个方法的最大问题就是，在使用本算法跨越封闭可及域时，可能会导致封闭可及域的变化，从而使得规划好的封闭可及域级路线出现变动。因此本算法有严格的适用范围，必须大概保证进行跨越之后，除了正在被抵达和玩家当前所在的可及域这两个封闭可及域之外，封闭可及域图中剩下的所有其他封闭可及域都不会随玩家穿越的行为而变化。对于小范围的跨越问题，建议直接用本算法进行搜索，不要先规划封闭可及域级路线了。
