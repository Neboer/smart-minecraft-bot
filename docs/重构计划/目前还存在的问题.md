## 第一阶段目标

1. Player应该可以走上一级台阶，而不是只能走平地被1层方块挡住。
2. 物品栏满了的时候只是不能添加新物品了。
3. AI完成的工作还不太完善，build_xxx_group 是否必要？还有一些Intent没有实现（而是实现在了player.py中）。比如 PlaceIntent。 build_mutation_group_sequence 作为Intent的方法非常不合理，比如 PlaceIntent 应该有自己的 param，比如 place_position ，然后它可以直接变成Mutation。
4. 有关PlayerID携带的问题：Intent中一律不带PlayerID，因为Intent必须由Player发出，它一定是与一个Player绑定的，只需要game.xxplayer.submitIntent(intent)就行了。 Intent 和 Player 是独立的（任何世界里的任何Player可以执行相同的Intent），但Mutation如果是 PlayerBaseMutation，就和Player是绑定的。
5. 程序里还有很多传字符串的地方不正确，应该改为传对象，枚举类型等等。程序返回值显得过于冗杂，只应该返回必要信息，像游戏刻这种不需要返回值的系统就不要返回值。
6. 目前我们只需要单玩家，但系统形式上应该支持多个玩家的情况。所以就直接假设player_id是唯一存在的并且在游戏里一定存在，后续如果需要多玩家了再改。

## 第二阶段目标

1. 玩家每tick可以按顺序提交多个Intent给game，只要保证这些Intent的总消耗tick正好小于1即可，game会按顺序执行这些Intent。有些Intent消耗0tick，比如NoIntent。目前这个设计用处不大，目前除了NoIntent以外的Intent都消耗1tick。
2. 部分结构存在list/sequence/iterable等类型混用的情况，请你检查并修复问题，尤其务必确保类型关系协变和逆变正确，修复如dig.py 56行、world 167行等地方的类型不匹配的错误。
3. 正确考虑 _ 开头的函数名是否真的是私有函数，是否需要改为公有函数修复命名，如player的_add_item_to_inventory方法。
4. 各种mutation里，比如SinglePlayerBaseMutation，构造的时候不需要传probability，一个确定了的mutation的概率一定是1。概率不是靠Mutation来记录的，而是Intent在制作MutationGroupSequence时，决定每个MutationGroup里Mutation的概率，目前dig.py中的写法是正确的。
5. PlaceBlockIntent必须接受一个position参数，指定放置方块的位置啊，不要通过 world.resolve_place_position 替玩家决定放置的位置，玩家应该在Intent里提交自己要放在哪里（三个合法放置位置：position、position high、below，玩家需要提交Vec3I表示这个目标位置。）
6. assert式断言，应该只能发生在与Player有关的功能中，用于表达“game中始终存在唯一的player”，不应该像finish_dig中assert block is not None、assert ok都是不行的，如果block是None、ok是false这种错误是需要处理的。程序总体来说除了唯一player不应该主动抛出异常。
